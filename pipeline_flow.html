Agent Memory Pipeline 流程图

这张图展示当前代码的完整链路：Build 先把历史会话建成 memory store； Answer 复用已有 store 做检索、时间线排序和回答；Judge / Metrics 在预测文件上继续评估。

输入 / 数据 Build / Store Answer Judge / Metrics 可选分支

总流程图

Build 与 Answer 的输入输出

Build 输入

  • --data：数据文件，例如 longmemeval_s_cleaned.json
  • --mode build 或 --mode full
  • --memory-mode raw/extract
  • --chunk-unit turn/pair/session

Build 输出

  • outputs/<run-name>/stores/...
  • chunks.json：chunk 文本和 provenance
  • embeddings.npy：向量矩阵
  • build_stats.json：tokens、chunk 数、耗时

Extract 输出结构

  • 一个 session 可抽多个 fact
  • 每个 fact 独立 embedding
  • 绑定 event_date、role
  • 其他顺序信息不再单独保存，避免 metadata 过重

Answer 输入

  • 当前 sample 的 question 和 question_date
  • 对应 memory_id 的已建 store
  • --top-k 控制召回条数
  • 不需要重新 build，除非 store 不存在

Answer 输出

  • predictions.jsonl
  • hypothesis / raw_response
  • retrieved_chunks：保存召回 memory 内容和 session_id
  • evidence_session_recall / all_found

当前可选方式

  • 运行：build、answer/query、full
  • memory：raw、extract
  • chunk：turn、pair、session
  • 后端：OpenAI API 或本地 vLLM

常用命令

目的 命令 主要输出
Build extract memory python run_baseline.py --config configs/vllm.yaml --dataset longmemeval --data ../LongMemEval/data/longmemeval_s_cleaned.json --mode build --run-name lme_s_extract --memory-mode extract --chunk-unit session outputs/lme_s_extract/stores/
Answer python run_baseline.py --config configs/vllm.yaml --dataset longmemeval --data ../LongMemEval/data/longmemeval_s_cleaned.json --mode answer --run-name lme_s_extract --memory-mode extract --chunk-unit session --top-k 20 outputs/lme_s_extract/predictions.jsonl
Judge python -m evaluation.judge --config configs/vllm.yaml --run-name lme_s_extract outputs/lme_s_extract/predictions.judge.jsonl
Metrics HTML python -m evaluation.metrics --run-name lme_s_extract --format html outputs/lme_s_extract/metrics.html
